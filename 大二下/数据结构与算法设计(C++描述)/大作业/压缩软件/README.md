# 哈夫曼编码文件压缩软件
## 一、项目简介
基于 C++ 哈夫曼编码算法的无损压缩工具，支持任意格式文件（文本、图片等）的压缩与解压，压缩后生成 `.huff` 文件，解压可精准还原原始文件，无失真。

## 二、运行方法
1. 环境：支持 C++11 及以上编译器（GCC、VS2022 等），跨平台兼容；
2. 编译：`g++ 压缩软件.cpp -o huffman_compressor -std=c++11`（无需额外依赖）；
3. 启动：运行可执行文件，按提示选择操作并输入文件名（支持带空格）。

## 三、文件说明
| 文件名         | 用途                                                         |
|----------------|--------------------------------------------------------------|
| 压缩软件.cpp   | 唯一源代码文件，整合编码逻辑、文件IO及控制流程，实现完整功能 |

## 四、核心类与函数用途
### 1. HuffmanNode（哈夫曼节点类）
- 用途：存储树节点的字符、频率及子节点指针，支撑哈夫曼树构建。

### 2. HuffmanTree（哈夫曼树管理类）
| 函数名                  | 用途                                                 |
|-------------------------|------------------------------------------------------|
| buildTree(freqMap)      | 基于字符频率表，通过优先队列合并节点构建哈夫曼树     |
| generateCodes(node, code) | 递归遍历树，生成字符-二进制编码映射                  |
| deleteTree(node)        | 递归释放节点内存，避免泄漏                           |
| getCodes()              | 返回字符-编码映射表                                  |

### 3. FileHandler（文件IO类）
| 函数名                  | 用途                                                 |
|-------------------------|------------------------------------------------------|
| readFile(filename)      | 二进制读取文件，统计字符频率并返回频率表              |
| writeEncoded(filename, codes) | 生成.huff压缩文件，写入编码表和压缩数据（8位/字节）  |
| decodeFile(filename)    | 读取.huff文件，重建解码表，还原原始文件（_decoded后缀） |

### 4. HuffmanCompressor（主控制类）
| 函数名                  | 用途                                                 |
|-------------------------|------------------------------------------------------|
| compress(filename)      | 压缩入口：读文件→统计频率→建哈夫曼树→生成压缩文件    |
| decompress(filename)    | 解压入口：读.huff文件→解码→生成原始文件              |

### 5. main()
- 用途：提供交互界面，接收用户操作（压缩/解压）和文件名，调用核心函数。

## 五、操作指南
| 步骤       | 操作说明                                                                 |
|------------|--------------------------------------------------------------------------|
| 选择功能   | 输入「1」压缩，「2」解压                                                 |
| 输入文件名 | 压缩：输入原始文件路径（含后缀，如`test.txt`）；<br>解压：输入.huff文件路径 |
| 查看结果   | 压缩生成「原文件名.huff」；解压生成「原文件名_decoded」（与原文件一致）   |

## 六、注意事项
1. 支持任意二进制文件，解压无失真；
2. 压缩文件固定后缀为`.huff`，解压需输入带该后缀的文件；
3. 支持256种字符（覆盖ASCII码），满足多数场景；
4. 已处理空文件、文件打开失败等异常，输出错误提示；
5. 跨平台兼容，无需系统特定API。
