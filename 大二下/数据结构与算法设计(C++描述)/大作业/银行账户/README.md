# 银行账户管理系统
## 一、项目概述
本系统基于C++语言开发，采用单向链表存储数据，支持客户与管理员双身份登录，实现开户、查询、存款、取款、转账、销户、账户浏览等核心功能，通过文件读写实现数据持久化，适配Windows环境下的命令行操作。

## 二、环境要求
- 操作系统：Windows 11
- 编译工具：Visual Studio 2022
- 依赖：标准C++库（无需额外第三方依赖）

## 三、文件说明（按功能分类）

| 文件名          | 用途                                                                 |
|-----------------|----------------------------------------------------------------------|
| BankAccount.h   | 声明账户类（BankAccount），存储账户ID、姓名、余额等信息及链表指针，友元客户/管理员类 |
| Clients.h       | 声明客户类（Clients），包含开户、登录、存取款等客户操作函数声明       |
| Manager.h       | 声明管理员类（Manager），包含浏览、查找、转账等管理员操作函数声明     |

### 2. 核心类实现（.cpp）：构造与析构
| 文件名          | 用途                                                                 |
|-----------------|----------------------------------------------------------------------|
| BankAccount.cpp | 实现账户类的默认构造和带参构造，初始化账户信息和链表节点             |
| Clients.cpp     | 实现客户类构造（初始化链表头节点）和析构（释放链表内存）             |
| Manager.cpp     | 实现管理员类构造，默认密码设为"66666666"                             |

### 3. 主程序入口
| 文件名                          | 用途                                                                 |
|---------------------------------|----------------------------------------------------------------------|
| Bank_Account_Management_System.cpp | 程序主函数，显示身份选择菜单，处理客户/管理员登录流程及功能跳转       |

|-----------------|----------------------------------------------------------------------|----------------------------------------------------------------------|
| open_accounts.cpp | 实现客户开户功能                                                     | 输入姓名、密码（两次验证）、ID等信息，自动生成16位银行卡号，插入链表 |

### 6. 数据持久化
| 文件名          | 用途                                                                 | 使用说明                                                             |
|-----------------|----------------------------------------------------------------------|----------------------------------------------------------------------|
| writein.cpp     | 从文件读取账户数据初始化链表                                         | 程序启动时调用，读取data.txt数据，恢复账户信息                       |
| reflesh.cpp     | 将账户数据写入文件保存                                               | 操作结束退出时调用，将链表数据写入data.txt，实现数据持久化           |

## 四、核心功能说明
1. **双身份登录**：客户（开户/存取款等）、管理员（浏览/查找等）权限分离
2. **数据管理**：单向链表动态增删账户，文件读写避免数据丢失
3. **异常处理**：支持密码不一致、余额不足、无效输入等场景提示

## 五、使用步骤
1. 编译所有源文件，生成可执行程序
2. 运行程序，选择身份（客户/管理员/退出）
3. 客户操作：可选择开户或登录，登录后办理存取款、转账等业务
4. 管理员操作：输入默认密码"66666666"登录，执行浏览、查找等操作
5. 退出程序时自动保存数据到data.txt

## 六、注意事项
1. 密码为明文显示，无登录失败次数限制，建议后续优化隐藏输入
2. 随机生成的16位银行卡号可能重复，需额外添加唯一性校验
3. 所有操作通过命令行完成，输入需符合提示格式（如金额为数字）
4. 首次运行无data.txt时，需先通过客户身份开户生成账户数据
